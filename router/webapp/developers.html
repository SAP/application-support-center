Developer Guide

ASC is built to provide in-app help, announcements, release notes and application management from a single platform. As developers, we were tired of building the same functionality within all of our apps, and decided it would be best to consolidate some of those features into a single SDK, which could be added to our source code. This provides our applications with a consistent look and feel across our portfolio, as well as reduces the amount of time and effort to get a new application released.

The process is extremely straight forward, applications are listed in the admin console and given a unique ID which is used to identify the application and consume the content from your end users perspective. As a developer, you can do 2 things. You can install the feedback service or you can download the relevant SDK, enter a few lines of code, compile (or deploy) and you are up and running.

We HIGHLY recommend all native apps use the feedback service. This service combines several frameworks. Feedback Shake, Global IT Tracking, ASC, Crashlogs, JamfKeychain and adds new features such as rating and suggest a feature. When you use Feedback Service there is no need to separately install libraries below like we used to. See JAM page.

You also have the option to install ASC via the SDK's or API's, although this option is not used much any longer.

Both the feedback service and the ASC-SDK have the following:

  • Release Notes
  • Help

Copy the code below to your js file from a press event (or however you prefer)

//Load SAP App Support Center, this should be in your Component.js file try { jQuery.sap.registerModulePath("sap.asc", "https://appsupport.services.sap/asc_ui5_lib"); jQuery.sap.require("sap.asc"); sap.asc.setHelpServer("https://appsupport.services.sap"); sap.asc.setAppId("(app_id)"); sap.asc.setAccessToken("(token)"); sap.asc.init(); sap.asc.open(); //Put this line into your action button which will open the ASC Dialog } catch (err) { jQuery.sap.log.error("Could not load ASC"); }

The Web UI for Application Service Center renders your content in a browser based interface. This can be included as a link from your application to open the web page in your device, or desktops, native browser.

https://appsupport.services.sap/portal/index.html?appid=(app_id)

If you prefer to consume the data in the backend without using a defined SDK, or want to use the data for other purposes, using the API is probably the best way to go. The service returns standard JSON. You will need to include your App Authentication Token.

//Retrieve the App Data set curl 'https://appsupport.services.sap/api/v1/apps/(app_id)?access_token=(token)' curl 'https://appsupport.services.sap/api/v1/apps/(app_id)/help?access_token=(token)' curl 'https://appsupport.services.sap/api/v1/apps/(app_id)/announcements?access_token=(token)' curl 'https://appsupport.services.sap/api/v1/apps/(app_id)/releases?access_token=(token)'
